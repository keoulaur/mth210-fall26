    Subsection What’s a relation?

    Here is the most general definition of relation (which is on page 393 of your textbook):
    🔗
    A relation from \(A\) to \(B\) is a subset \(\sim\) of \(A\times B\text{.}\)
    🔗
    A relation on \(A\) is a subset \(\sim\) of \(A\times A\text{.}\)
    🔗
    Recall the Cartesian product of \(A\) and \(B\) is denoted \(A\times B\) and means the ordered pairs where the first element is in \(A\) and the second element is in \(B\text{.}\) E.g.,
    \begin{equation*} A\times B = \{ (a,b): a\in A \text{ and } b\in B\}. \end{equation*}
    🔗
    🔗

    Subsection Representations and Examples

    A relation can be written as
    🔗
    • a ``rule’’ (typically an if or an if and only if statement),
      🔗
      🔗
    • a set of ordered pairs (either in set-builder or roster notation)
      🔗
      🔗
    • a directed graph
      🔗
      🔗
    Example: In the first example in Chapter 9, the author defines a relation \(A = \mathbb{N}\) where we say \(a\sim b\) if \(a\geq b\text{.}\) Here’s 3 different ways to write that relation on \(\mathbb{N}\text{:}\)
    🔗
    • \(a\sim b\) if and only if \(a\le b\)
      🔗
      🔗
    • \(\displaystyle \{(a,b)\in\mathbb{N}\times\mathbb{N}: a\le b\}\)
      🔗
      🔗
    • Notice that it’s hard to write the set form in roster notation, we might write something like
      \begin{equation*} \{(1,2), (1,3), (1,4),\dots, (2,3), (2,4), (2,5), \dots, (3,4), (3,5), (3,6),\dots \} \end{equation*}
      but even then it might be hard for our reader to see what’s going on.
      🔗
      🔗
    • We can’t draw the graph neatly on infinitely many things, but here’s one if the relation were on \(\{1,2,3,4,5\}\text{:}\)
      🔗
      🔗

    Subsection More Definitions!

    Some relations on a set \(A\) are special because they act like ``=’’ (and, IMHO, = is one of the most important symbols in all of math!). They have the following the properties:
    🔗
    • Reflexive: For all \(a\in A\text{,}\) \(a\sim a\)
      🔗
      🔗
    • Symmetric: For all \(a,b\in A\text{,}\) if \(a\sim b\) then \(b\sim a\text{.}\)
      🔗
      🔗
    • Transitive: For all \(a,b,c \in A\text{,}\) if \(a\sim b\) and \(b\sim c\) then \(a\sim c\text{.}\)
      🔗
      🔗
    If all three of the properties hold, we call the relation an equivalence relation.
    🔗

    Conjecture 53.

    Define the relation \(\sim\) on \(\mathbb{R}\) as follows: for all \(a,b\in \mathbb{R}\text{,}\) \(a\sim b\) if and only if \(ab\geq 0\text{.}\) Then \(\sim\) is reflexive, symmetric, and transitive.

    Let \(U\) be some nonempty universal set and define the relation \(\sim\) on \(\mathcal{P}(U)\) as follows: for all \(A,B\in \mathcal{P}(U)\text{,}\) \(A\sim B\) if and only if \(A\cap B = \emptyset\text{.}\) Then \(\sim\) is reflexive, symmetric, and transitive.

    Subsection Equivalence Classes

    Definition 55.

    Let \(\sim\) be an equivalence relation on a nonempty set of \(A\text{.}\) Then the equivalence class of of \(a\) determined by \(\sim\) is
    \begin{equation*} [a]_{\sim} = \{x\in A \mid x \sim a \} . \end{equation*}
    🔗
    🔗

    Theorem 56. Equivalence Classes Partition.

    [Equivalence Classes Partition] Let \(A\) be a nonempty set and let \(\sim\) be an equivalence relation on \(A\text{.}\) Then
    🔗
    1. For each \(a\in A\text{,}\) \([a]\) is nonempty (in particular, \(a\in [a]\)).
      🔗
      🔗
    2. For each \(a,b\in A\text{,}\) \(a\sim b\) if and only if \([a]=[b]\text{.}\)
      🔗
      🔗
    3. For each \(a,b\in A, [a] =[b]\) or \([a] \cap [b] = \emptyset\text{.}\)
      🔗
      🔗
    🔗

    Subsection Congruence Classes and Congruence Arithmetic

    If the equivalence relation is congruence, then we use the term congruence class rather than equivalence class. The great thing about congruence classes is that you can add and multiply them in a natural way.
    🔗

    Definition 57.

    Let \(a,b\in\mathbb{Z}\) and \(n\in\mathbb{N}\text{.}\) Then
    🔗
    \([a]_n + [b]_n = [a+b]_n\) and \([a]_n\cdot [b]_n = [a \cdot b]_n\text{.}\)
    🔗
    🔗
    This may seem obvious to you, but there are some very subtle things going on. Perhaps the weirdest is that \([a]_n\) is really a set, an infinite set. So what you’re actually doing is adding and multiplying infinite sets.
    🔗
    Example: What is \([1]_3 +_3 [2]_3\text{?}\) What is \([2]_3 \cdot_3 [2]_3\text{?}\)
    🔗
    Notice that we’ve only used the congruence classes \([0]_3, [1]_3,\) and \([2]_3\) since these are the only distinct congruence classes modulo \(3\text{.}\) We typically let \(\Z_3 = \{[0]_3, [1]_3,[2]_3\}\text{.}\) Also, note \([2]_3 \cdot [2]_3 = [4]_3\text{.}\) However, \([4]_3=[1]_3\) so we choose \(1\) to be our representative of that congruence class.
    🔗
    Here are are addition and multiplication tables when we consider congruence modulo \(3\text{:}\)
    🔗
    \(+_{3}\) \([0]_{3}\) \([1]_{3}\) \([2]_{3}\)
    \([0]_{3}\) \([0]_3\) \([1]_3\) \([2]_3\)
    \([1]_{3}\) \([1]_3\) \([2]_3\) \([0]_3\)
    \([2]_{3}\) \([2]_3\) \([0]_3\) \([1]_3\)
    \(\cdot_{3}\) \([0]_{3}\) \([1]_{3}\) \([2]_{3}\)
    \([0]_{3}\) \([0]_3\) \([0]_3\) \([0]_3\)
    \([1]_{3}\) \([0]_3\) \([1]_3\) \([2]_3\)
    \([2]_{3}\) \([0]_3\) \([2]_3\) \([1]_3\)
